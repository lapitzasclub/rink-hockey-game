# .copilot

Esta carpeta es la memoria operativa e indexada del proyecto.

## Regla principal para cualquier agente

La primera instrucción de cualquier agente que trabaje en este proyecto es:

> Mantén esta carpeta actualizada con información veraz, concreta e indexada sobre el proyecto según avances.

Eso implica:

- Actualizar documentación cuando cambie arquitectura, alcance, decisiones o estado.
- No inventar hechos ni dejar documentación desfasada.
- Registrar decisiones importantes y por qué se tomaron.
- Mantener un índice claro para que otro agente pueda retomar el trabajo rápido.

## Estructura

- `project-overview.md`: visión general, objetivo y alcance.
- `architecture.md`: tecnologías, estructura técnica y decisiones de implementación.
- `roadmap.md`: hitos, backlog y estado de avance.
- `assets-index.md`: inventario de sprites, audio, animaciones y su estado.
- `game-design.md`: reglas, controles, mecánicas y comportamiento del juego.
- `work-log.md`: cambios realizados en orden cronológico.
- `agent-instructions.md`: instrucciones permanentes para agentes colaboradores.

## Convenciones

- Escribir en español claro.
- Priorizar información accionable.
- Fechar decisiones relevantes.
- Si algo es provisional, marcarlo explícitamente como provisional.
